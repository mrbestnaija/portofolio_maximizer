## Rule

Refer broadly, update narrowly.

- Read the authoritative document for a subsystem before making claims.
- Update only the source-of-truth document for the behavior you changed.
- Use temporary handoff notes only for multi-agent coordination or bounded implementation slices.

## Update Rules

1. `PROJECT_STATUS.md`
- runtime truth only
- every material claim should trace to a command or artifact

2. `AGENT_*` temporary docs
- must carry metadata header fields from `DOC_METADATA_CONTRACT.md`
- must state owner, supersedes target, and expiry condition

3. generated snapshots
- must be regenerated by script, not edited by hand

4. experiment docs
- may track research hypotheses and results
- must not make readiness or gate-promotion claims unless the canonical status docs also support them

## Non-Goals

- This file does not replace subsystem design docs.
- This file does not grant authority to temporary docs over canonical runtime/policy documents.
