# Logs

This directory stores runtime artifacts, run outputs, and operational traces.

Most generated files under `logs/` are ignored by design. Use the docs below for the canonical
index and retention rules:

- [Documentation/LOG_ORGANIZATION_SUMMARY.md](../Documentation/LOG_ORGANIZATION_SUMMARY.md)
- [Documentation/history/README.md](../Documentation/history/README.md)

Common log families live in subdirectories such as `audit_gate/`, `automation/`, `errors/`,
`events/`, `forecast_audits/`, `observability/`, `run_audit/`, `signals/`, `stages/`, and
`warnings/`.
